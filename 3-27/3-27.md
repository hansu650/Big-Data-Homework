Chapter 3 到底在讲什么

第三章标题叫 Data Preprocessing，也就是数据预处理。
PPT 一上来就把这一章的大框架列出来了：先讲 Data Quality（数据质量），再讲几个主要任务：Data Cleaning、Data Integration、Data Reduction、Data Transformation and Data Discretization。

你先不要急着记这些英文名。
你先抓住一句最核心的话：

第三章其实就在讲：正式做数据挖掘之前，得先把数据收拾好。

一、为什么要先做预处理

PPT 这里先问了一个问题：

Why Preprocess the Data?

也就是：

“为什么数据不能直接拿来用？”

因为现实中的数据，很多时候并不是你想象中那种整整齐齐、干干净净、完全正确的数据。

老师这里其实是在提醒你：

数据可能有错
数据可能缺东西
数据可能前后矛盾
数据可能太乱
数据可能格式都不统一

所以，不是拿到数据就能直接跑算法。
很多时候真正花时间的不是“建模”，而是前面的“收拾数据”。

你可以把它想成：

算法像做饭
数据像食材
第三章讲的就是：洗菜、切菜、挑烂菜、分装、去杂质

如果食材本身就烂了，后面火候再好也没用。

二、什么叫 Data Quality（数据质量）

PPT 说，数据质量要从多个维度来看，也就是 a multidimensional view。
它列了 6 个词：Accuracy、Completeness、Consistency、Timeliness、Believability、Interpretability。

我们一个一个慢慢翻。

1. Accuracy = 准确性

PPT 的原话很简单：
correct or wrong, accurate or not。

意思就是：

这个数据到底对不对。

比如一个人的工资写成 -10，那明显就不对。
比如一个学生成绩写成 1000，也很可能录错了。

所以“准确性”就是在看：
这条数据是不是符合真实情况。

你可以把它理解成：

这不是“有没有”，而是“真不真”。

2. Completeness = 完整性

PPT 写的是：
not recorded, unavailable。

意思就是：

该有的数据有没有缺。

比如学生表里本来应该有：

学号
姓名
年龄
专业
成绩

结果有些人的“专业”是空的，有些人的“成绩”没填。
那这就是不完整。

所以完整性关注的是：

数据是不是缺字段、缺值、缺记录。

3. Consistency = 一致性

PPT 这里写的是：
some modified but some not, dangling。

这个词很多同学第一次看会有点懵。
你可以把它理解成：

数据前后能不能对得上，彼此有没有打架。

举个最简单的例子：

年龄写 42
出生日期写 2010 年

这俩明显对不上。
那这就是不一致。

再比如一个系统里用户性别写 M/F，另一个系统写 男/女，再另一个写 0/1。
它们未必错，但会造成混乱，也属于一致性问题。

所以一致性核心是：

同一个意思，在不同地方是不是统一；同一条数据内部有没有矛盾。

4. Timeliness = 时效性

PPT 写的是：
timely update?

意思是：

数据是不是够新，更新得及时不及时。

比如一个人的住址已经变了，但系统还存着三年前的地址；
库存早就卖空了，系统里还显示“有货”。

这种问题不是“错得离谱”，而是过期了。

所以时效性就是：

数据有没有跟上现实世界的变化。

5. Believability = 可信度

PPT 写的是：
how trustable the data are correct?

这个和准确性很像，但不完全一样。

准确性 更偏“它实际上对不对”
可信度 更偏“你敢不敢信它”

比如两份数据都写了用户年龄：

一份来自身份证实名信息
一份来自用户自己随手乱填

那显然前者可信度更高。

所以可信度关注的是：

数据来源靠不靠谱，值值不值得相信。

6. Interpretability = 可解释性 / 可理解性

PPT 写的是：
how easily the data can be understood?

这个意思是：

别人拿到这份数据，能不能看懂。

比如有个字段叫：

x1, x2, x3

但是完全没有说明。
那别人就不知道它代表什么。

或者一个字段混着多种编码方式，也会让人难以理解。

所以可解释性就是：

数据有没有清楚的含义，格式和命名是不是容易读懂。

三、这 6 个词你怎么整体理解

你别把这 6 个词分开死背。
你可以把它们连成一句非常口语的话：

一个好数据，应该是对的、全的、不打架的、没过期的、能信的、看得懂的。

对应就是：

对的 → Accuracy
全的 → Completeness
不打架 → Consistency
没过期 → Timeliness
能信 → Believability
看得懂 → Interpretability

这样记就顺很多。

四、第三章接下来主要干什么

在讲完“数据质量”后，PPT 马上列出了 Major Tasks in Data Preprocessing，也就是“数据预处理的主要任务”。
它分成四大块：Data cleaning、Data integration、Data reduction、Data transformation and data discretization。

我们先不深讲，只先建立感觉。

1. Data cleaning = 数据清洗

PPT 对它的说明是：

填补缺失值
平滑噪声数据
识别或删除离群点
解决不一致问题

你可以简单理解成：

把脏数据处理干净。

比如：

空值补一下
明显错的数据修一下
很离谱的异常值检查一下
编码不统一的字段统一一下

这个是第三章里最接地气的一块。

2. Data integration = 数据集成

PPT 说的是：

Integration of multiple databases, data cubes, or files。

也就是：

把多个来源的数据合起来。

比如：

一个 Excel 是学生基本信息
一个数据库是成绩
一个系统里是选课记录

你要把它们合成一份统一数据，这就叫集成。

这里的难点一般不是“拼起来”，而是：

字段名不一样
格式不一样
同一个人可能重复出现
同一个属性不同系统单位不同

这个后面 PPT 会细讲。

3. Data reduction = 数据规约 / 数据缩减

PPT 提到这块包括：

Dimensionality reduction
Numerosity reduction
Data compression

你可以把它理解成：

让数据变小一点，但别把有用信息弄丢太多。

比如：

原来有 100 个特征，删成 20 个
原来 1000 万条数据，抽样成 10 万条
原来很大，先压缩一下

为什么要这么做？
因为数据太大，算法跑得慢，甚至跑不动。

4. Data transformation and data discretization = 数据变换与离散化

PPT 这里列了两个例子：

Normalization
Concept hierarchy generation

这部分你现在先只要有个感觉：

数据变换

就是把数据改成更适合后续分析的样子。
比如：

把很大的数缩放到 0 到 1
把多个字段组合成一个新字段
数据离散化

就是把连续值切成区间。
比如年龄原来是具体数字：

18
19
20
21

后来改成：

青年
中年
老年

这就叫离散化。

五、你现在先不要背后面细节，只要先建立这一张图

你可以先把第三章开头记成这样：

第三章的逻辑
为什么要预处理？
因为原始数据质量不一定好。
什么叫数据质量？
看它准不准、全不全、一不一致、新不新、可不可信、好不好懂。
预处理主要做什么？
清洗
集成
缩减
变换/离散化
六、我帮你用最像考试的话说一遍

如果老师问：

“Why preprocess the data?”

你可以答：

因为现实中的原始数据通常存在质量问题，比如不准确、不完整、不一致、不及时、不够可信或不易理解，因此需要先进行数据预处理，再进行后续的数据挖掘与分析。
